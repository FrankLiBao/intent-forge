# 安全政策

感谢您帮助 Intent Forge 对每个人都保持安全。

Intent Forge 非常重视我们软件产品和服务的安全性，包括通过我们的开源代码仓库管理的所有代码。

我们目前支持以下版本的安全更新：

| 版本 | 支持状态 |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## 报告安全问题

如果您认为在 Intent Forge 仓库中发现了安全漏洞，请通过协调披露向我们报告。

**请不要通过公开的 GitHub issues、讨论或拉取请求报告安全漏洞。**

相反，请发送电子邮件至项目维护者。您可以通过以下方式联系：

- 在仓库中创建私密安全咨询（Security Advisory）
- 或通过项目仓库的联系方式与维护者联系

请尽可能包含以下信息，以帮助我们更好地理解和解决问题：

- 问题类型（例如，缓冲区溢出、SQL 注入或跨站脚本）
- 与问题表现相关的源文件的完整路径
- 受影响源代码的位置（标签/分支/提交或直接 URL）
- 重现问题所需的任何特殊配置
- 重现问题的分步说明
- 概念验证或漏洞利用代码（如果可能）
- 问题的影响，包括攻击者如何利用该问题

此信息将帮助我们更快地分类您的报告。

## 安全响应流程

1. **确认收到**：我们将在 48 小时内确认收到您的报告
2. **初步评估**：我们将在 7 天内提供初步评估
3. **修复开发**：根据严重性，我们将制定修复计划
4. **披露时间表**：我们将与您协调披露时间表
5. **发布修复**：我们将发布安全更新并归功于发现者（如果您希望）

## 安全最佳实践

使用 Intent Forge 时，我们建议：

- 始终使用最新版本的 Intent Forge CLI
- 定期更新依赖项：`uv sync --upgrade`
- 审查 AI 生成的代码是否存在安全问题
- 不要在规范或配置文件中提交敏感信息（密钥、令牌等）
- 使用 `.gitignore` 排除包含凭据的文件
- 在生产环境中部署前审查所有生成的代码

## 已知安全考虑

使用 AI 代理生成代码时：

- **代码审查**：始终审查 AI 生成的代码，特别是涉及身份验证、授权或数据处理的代码
- **依赖项**：检查生成的依赖项是否存在已知漏洞
- **输入验证**：确保生成的代码正确验证用户输入
- **敏感数据**：避免在提示或规范中包含敏感信息，因为它们可能被记录
